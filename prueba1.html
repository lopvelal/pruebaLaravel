Task description

You are given a blogging application in Laravel. Define the routes needed for the blog.

The application is split into two parts, for guests and for administrators, and contains the following functionalities:

For guests:

  • Show a list of titles of the most recent posts
  • Show a single post

For administrators:

  • Show a list of titles of the most recent posts with edit and delete buttons
  • Add a new post
  • Edit a post
  • Delete a post

Requirements

Your task is to create a full routes/web.php file. Assume that the rest of the application is fully implemented.

  • The part for guests should be open to any user and should have no prefix
  • The part for administrators should be restricted to logged-in users, using Laravel's standard authentication, and should have an admin prefix

Correct HTTP methods should be used:

  • get to retrieve resources
  • post to create resources
  • put to update resources
  • delete to remove resources

The routes should be named:

  • site.post.index
  • site.post.show
  • admin.post.index
  • admin.post.show
  • admin.post.store
  • admin.post.update
  • admin.post.delete

The routes should point to the following URLs respectively:

  • /
  • /post/<id>
  • /admin/post
  • /admin/post/show/<optional_id>
  • /admin/post/store
  • /admin/post/update
  • /admin/post/delete/<id>
